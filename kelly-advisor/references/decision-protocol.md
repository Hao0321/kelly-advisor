# 決策協定 — 七步流程 + 分類決策樹 + 實戰範例

> SKILL.md 的操作手冊。skill 跑分析時照這個走。

---

## A. 分類決策樹（第一件事）

```
使用者描述一個「可能賺可能賠」的決策
        │
        ▼
這個決策「同一類」會在未來重複很多次嗎？
   │                          │
  否（樣本≈1）               是
   │                          │
   ▼                          ▼
不可逆嗎？/ 機率真的       要在多個並行機會間
能估嗎？/ 主要是錢嗎？      分配有限資本或時間嗎？
   │            │              │              │
不可逆/不可知/  可逆+可估+    是              否（單一機會）
非金錢          純金錢        │              │
   │            │            ▼              ▼
   ▼            ▼         Mode 2          Mode 1
 Mode 3       Mode 1      Allocate         Size
 Survive      Size
```

**判斷提示**：
- 「換工作 / 跳槽 / 創業 all-in / 搬家 / 要不要分手 / 要不要生小孩」→ 幾乎都是 **Mode 3**。
- 「這檔股票部位開多大 / 這個 SKU 廣告加碼多少 / 撲克這手下多少」→ **Mode 1**。
- 「我有 100 小時 / 50 萬，要分給 7 個產品怎麼分」→ **Mode 2**。

**⚠️ All-in tie-breaker（v0.1.1，dogfood 抓到）**：
同一個決策會因「framing」在 Mode 2 和 Mode 3 之間跳。判準：
- 「把時間**分給**多個專案」→ 可調整、可逆 → **Mode 2**。
- 「把時間**全押**一個、**停掉**其他」→ all-in framing 消滅了可逆性 → **Mode 3**，拒給數字。
- 規則：**句子裡有「all-in / 全押 / 梭哈 / 停掉其他」→ 強制 Mode 3**，先跑破產檢查，
  再把「要不要這樣賭」丟回 genius-advisor。不要因為它「看起來像配置題」就給 Kelly 數字。

## B. Mode 1 · Size（可重複下注 → 算 fractional Kelly）

走 SKILL.md 七步。

> **⚠️ 先定義 bankroll（v0.1.1，dogfood 抓到）**：凱利的「下注比例」是「佔 bankroll 的 %」，
> 不是「佔總資產 / 淨值的 %」。**bankroll = 你會拿來反覆下這類注的那個資金池**
> （例：投資 = 你的投資組合本金；廣告 = 你的行銷預算，不是公司全部現金；撲克 = 牌桌資金）。
> 套錯基數 = 算出來的部位差好幾倍。先講清楚「f% 是佔哪個池子」，再給數字。

產出格式：

```
【分類】Mode 1（可重複下注）
【bankroll】= ___（這類注的專用資金池，非總資產）
【破產檢查】最壞情況：___；會不會出局：否（已確認本金是 surplus）
【三要素】p ≈ __（估計，信心：低/中/高）｜賠率 b ≈ __｜本金（surplus，佔 bankroll）= __
【full Kelly】f* = __（≤2 位數）
【建議下注】½ f* = __  ← 用這個
【壓力測試】若 p 高估 15% → f* 變 __（仍正/轉負）
【一句洞察】（可截圖）例：「你以為該重押，凱利說只押 __%——因為你對自己的勝率沒那麼確定。」
```

## C. Mode 2 · Allocate（組合 / 配置）

多個並行押注，分配有限資本 / 時間。原則：
1. 每個機會各自算 `f*`（edge / 賠率）。
2. 負 edge 的直接 0（不分配）。
3. 正 edge 的按 `f*` 相對權重分配，但 **任何單一押注設硬上限**（避免單點毀滅）。
4. 全部一樣打折（½ 或更低）。

> 對 Hao 的場景：7 個產品的時間配置 = 按「每個產品的 edge（轉換 × 流量 × 你的 leverage）」相對加權，
> 但不把所有時間壓在一個（即使它 edge 最高）——留冗餘，防單一產品崩掉。

## D. Mode 3 · Survive（一次性 / 不可逆 → 拒給數字）

**這是這個工具最重要的模式。** 不算凱利。動作：

```
【分類】Mode 3（一次性 / 不可逆 / 機率不可知 / 非金錢效用）
【為什麼不算凱利】樣本=1、不可回頭、p 是猜的、含非金錢價值 → 凱利的前提不成立
【改問破產問題】
  1. 最壞情況是什麼？
  2. 如果最壞發生，你會「出局」嗎？（財務歸零 / 不可恢復 / 跑道燒完）
  3. 你的「跑道」有多長？（撐幾個月不死）
  4. 有沒有 barbell 版本？（90% 保命 + 10% 賭，而不是 all-in）
【路由】這題的「值不值得做 / 方向 A vs B」交給 genius-advisor 圓桌
       （個人決策 → Naval / Munger / Buffett；投資/資產 → Buffett / Munger / Kiyosaki）
```

**核心台詞**：「這不是下注題，是不可逆的人生決策。凱利會給你一個假裝精確的數字、害你過度自信。
我不給數字——我幫你確認最壞情況死不死得了，然後把『值不值得』交給圓桌。」

## E. 實戰範例庫

### 範例 1 — 投資部位（Mode 1，經典）
> 「我有個會反覆做的策略，勝率我估 60%，賺賠 1:1，部位開多大？」

```
分類：Mode 1。破產檢查：本金是可承受歸零的投資金 ✅
三要素：p≈0.6（信心：中）｜b=1｜本金=surplus
full Kelly：f* = 0.6 − 0.4/1 = 0.20
建議：½ × 0.20 = 10%
壓力測試：p 其實只有 0.52 → f* = 0.04 → 你「中等信心」的代價是部位差 5 倍
一句洞察：「你估 60% 勝率就該押 10%，不是梭哈。而且你對 60% 有多確定？沒那麼確定 → 押更少。」
```

### 範例 2 — 7 個產品時間配置（Mode 2）
> 「我這個月 100 小時，要分給 7 個 Gumroad 工具，怎麼分？」

```
分類：Mode 2。各算相對 edge（近期轉換 × 流量趨勢 × 你的 leverage / 興趣）。
規則：edge 負的（沒人買、你也不想做）→ 0 小時。
     正的按 edge 加權，但單一產品上限設 ~40%（不把月份壓死在一個）。
     留 ~15% 冗餘給「下一個新工具」（你的資產生產線，別讓它枯竭）。
一句洞察：「把時間當本金。最會賺的那個也別超過 40%——你要的是一條產品線，不是一張彩券。」
```

### 範例 3 — 換工作（Mode 3，英雄案例：拒答）
> 「我在猶豫要不要離職去接一個全職機會，用凱利算一下值不值得。」

```
分類：Mode 3。→ 拒絕用凱利。
為什麼：這是樣本=1、不可逆、機率你根本不知道、而且一大半是非金錢（成長、自由、關係）。
        硬塞 p 和賠率進公式 = 用數學的權威感包裝瞎猜，害你過度自信。
改問：離職後跑道多長（存款撐幾個月）？最壞情況（新工作崩了）你死得了嗎？
     有沒有不 all-in 的版本（談 remote / 兼職過渡 / 留退路）？
路由：「值不值得跳」交給 genius-advisor 圓桌（Naval 長期遊戲 + Munger 反向掃雷 + Buffett 不可逆決策）。
一句洞察：「凱利公式拒絕回答這題——這正是它幫你的方式。先確認你跳了餓不死，再讓圓桌談值不值得。」
```

### 範例 4 — all-in 創業（Mode 3）
> 「我想把全部積蓄 all-in 這個創業。」

```
分類：Mode 3（非遍歷、不可恢復）。
台詞：「all-in 是跟『你只活一次』對賭。100 個人 all-in、10 個發財，群體數據很美，
       但你是 1 個人，破產了就出局，沒有第二局。」
改成 barbell：90% 保住生存底線，10% 用在最激進的賭注（虧光也死不了）。
路由：圓桌談這個創業本身該不該做、怎麼做（Musk 第一原理 + Naval 槓桿 + Munger inversion）。
```

### 範例 5 — SKU 廣告加碼（Mode 1）
> 「某個工具廣告 ROAS 不錯，要從行銷預算加碼多少？」

```
分類：Mode 1（可重複、可逆、純金錢）。
三要素：把 ROAS 換算成賠率 b；p = 這波投放維持表現的機率（估計）；本金 = 行銷 surplus（非營運命脈）。
算 f* → 打折 → 壓力測試（ROAS 衰退 20% 會怎樣）。
一句洞察：「廣告加碼是少數真能用凱利的決定。但用『會衰退的賠率』算，不是用昨天的好成績算。」
```

## F. 反例庫（這些不能做）

| ❌ 錯誤示範 | ✅ 正確 |
|---|---|
| 「換工作 p=0.7 賠率 3:1，凱利說 all-in」 | Mode 3，拒答，跑破產檢查 + 路由圓桌 |
| 「f*=0.234，建議下注 23.4%」 | 「約 ½ Kelly ≈ 10%，且這建立在你猜的 p 上」 |
| 「正 edge，照 full Kelly 全押」 | 永遠 ½ 或更低 + ruin check |
| 「把生活費當本金算凱利」 | 生存資本 f=0，只賭 surplus |
| 「這檔幣 p=0.8，幫我算該買多少」 | 不報標的、不替你估別人資產的 p；只教你自己算 + 防破產 |
| 「我要 all-in X、停掉其他」當配置題給數字 | all-in framing → 強制 Mode 3，拒給數字（v0.1.1） |
| 算「佔總資產幾 %」 | 先定義 bankroll = 該類注的專用資金池，f% 佔它（v0.1.1） |

## G. Dogfood 紀錄（v0.1.1，2026-05-31）

第一次實測（3 mode）抓到 2 個設計漏洞，已修進規則：
1. **Mode 2/3 framing 跳動**：「把時間全押一個、停掉其他」表面像配置題（Mode 2），實則 all-in（Mode 3）。
   → 加 all-in tie-breaker（SKILL.md 路由段 + 本檔 A 段 + 反例庫）。
2. **bankroll 基數模糊**：非賭博情境（廣告）算「押幾 %」時沒定義佔哪個池子。
   → Mode 1 產出格式加「先定義 bankroll」步驟。
通過：Mode 1 廣告加碼（ROAS→賠率）、Mode 3 拒答（all-in 創業 / 換工作）數學與路由皆正確。
